Getting started

Everything on the website is managed from one place: the Ghost admin panel. You log in once and from there you can add or change any content.

Logging in

  1. Open your web browser and go to your site address followed by /ghost/ — for example https://yoursite.com/ghost/.
  2. Enter the email and password for your account.
  3. You arrive at the Dashboard. The dark menu on the left is how you move around.

The left-hand menu

Menu itemWhat it is for
DashboardOverview of the site. You rarely need this for content.
PostsMost content lives here: success stories, blog posts, events, newsletters, testimonials, activities, and the small "source" posts that feed numbers and tables.
PagesThe fixed pages: Home, About, Programs, Donate, Contact, and so on.
TagsThe labels that decide where a post appears. Usually created automatically as you type them.
MembersNot used for the public website content.
Settings (gear, bottom-left)Site name, logo, social links, navigation menu, and the theme's site-wide values.
Golden rule: a Post is a single item that belongs to a list (one story, one event, one newsletter). A Page is a whole standalone page of the site (About, Donate, Contact). When in doubt, almost everything you add day-to-day is a Post.
2

How the website is organised

The site is assembled from three things: tagged posts, a few pages, and some site-wide settings. Understanding the four ideas below makes everything else simple.

1. Tags decide where a post shows up

A post does not appear anywhere until you give it the right tag. The tag is the address that tells the website which section to display it in. For example, a post tagged success-stories appears in the Success Stories sections; a post tagged event appears on the Events page.

The first tag is the category. If a post has several tags, the first public tag is treated as its category and is shown as the little label on its card.

2. The same fields are reused everywhere

Every post and page has the same set of fields. This site reuses them in specific ways, so it helps to know what each one becomes.

FieldWhere to find itWhat it becomes on the site
TitleTop of the editorThe heading, a person's name, or a year
ExcerptPost settings → ExcerptThe short line on a card — and the event date for events
Feature imagePost settings → Feature imageThe card image, photo, or background
BodyThe main editing areaThe full content; some sections read specific lines or blocks from it
Meta titlePost settings → Meta dataA small label/heading on certain cards
Meta descriptionPost settings → Meta dataLink text / supporting line on certain cards
Canonical URLPost settings → Meta dataA link destination on certain cards
Facebook titlePost settings → Facebook cardA link/button URL (hero button, event "Register")
TagsPost settings → TagsWhich section the post appears in

3. Order is controlled by publish date

Within any section, items are ordered by their publish date (newest or oldest, depending on the section). To move an item up or down, change its publish date in Post settings → Publish date.

4. Fallback content

Several sections ship with built-in placeholder content. The moment you add your own (the right tagged post or table), your content replaces the placeholder automatically. Remove it and the placeholder returns — the site is never blank.

3

Everyday tasks

The handful of routines below cover almost everything you will do. Learn these once and the page-by-page reference in Section 5 becomes quick to follow.

Create a post (the most common task)

Use this for a success story, a blog post, an event, a testimonial, a programme activity, a newsletter, and the small "source" posts that feed numbers and tables.

  1. In the left menu, click Posts, then the New post button (top right).
  2. Type the Title at the top.
  3. Write or paste the body in the large area below the title (story text, photos, a video, a PDF, or a table — depending on the section).
  4. Open Post settings by clicking the gear icon at the top right.
  5. Under Tags, type the section's tag and press Enter (e.g. success-stories). Create it if it does not exist yet.
  6. Set the Excerpt (the short card line — or the event date, where noted).
  7. Set the Feature image (the card photo).
  8. Add any extra fields the section needs (Meta data, Facebook title) — see Section 5.
  9. Click Publish (top right) → Publish again to confirm.

Create a page and choose its template

Pages already exist for this site. You only do this when building a brand-new page (for example, adding a fourth programme).

  1. Click Pages → New page.
  2. Type the Title.
  3. Open Page settings (gear icon) → Template → choose the matching template (see the table in Section 9).
  4. Open Page URL and set the slug (the part of the address after the domain, e.g. sevadhama).
  5. Add the body content the page uses (see Section 5).
  6. Click Publish.
About slugs. The slug is the page's web address. Keep it short, lowercase, and use hyphens instead of spaces — e.g. get-involved. Changing a slug changes the page's link, so set it once and leave it.

Edit existing content

  1. Go to Posts or Pages and click the item in the list.
  2. Make your changes.
  3. Click Update (top right). Changes are live immediately.

Schedule something for later, or set its order

  1. In Post settings → Publish date, set the date and time.
  2. A future date schedules the post (it goes live automatically at that time and stays hidden until then).
  3. To reorder items inside a section, give them earlier or later publish dates relative to each other.
Events are different. Do not use a future publish date for an upcoming event, or it will stay hidden. Publish it now and put the event date in the Excerpt instead (see Section 7).

Add tags

  1. Open Post settings → Tags.
  2. Type the tag name and press Enter. Use the exact tag from Section 9 — spelling and hyphens matter.
  3. The first tag in the list becomes the post's category label, so order tags with the category first.
Public vs internal tags. A normal tag (e.g. arogyam-impact) is what the site looks for. A tag that starts with a hash, like #hidden, is internal and will not be matched — avoid the hash for section tags.
4

Adding rich content

Inside the body of any post or page you can add more than text. Put your cursor on a new line and click the + button (or type /) to open the content menu.

Image

  1. On a new line, click + → Image and upload the photo.
  2. To add a caption (used on the Awards page), click "Type caption…" under the image and write the text.

Video

  1. Paste a YouTube link on its own line — Ghost turns it into a player automatically; or
  2. Use + → Video to upload a video file from your computer.
On a success story, adding a video makes a play button appear on its card. Clicking the play button opens the video in a popup, while "Read story" still opens the full post.

PDF (newsletters and annual reports)

  1. On a new line, click + → File and upload the PDF.
  2. The card automatically shows the PDF's first page as the cover image — you do not need a feature image.

Gallery (home and programme galleries)

  1. On a new line, click + → Gallery and drop in several images.
  2. A single gallery holds up to 9 images. To add more, add a second gallery below the first — all images merge into one gallery on the page.

Table (statistics, sponsorship, legal)

Ghost has no built-in table button, so add one with a Markdown card (+ → Markdown) using the simple format below.

// A markdown table — the first row is the header, then one row per item
| id | name | number |
|----|------|--------|
| wheelchair | Wheelchairs Distributed | 438 |
| ambulance  | Ambulance Services      | 403 |

Each | separates a column. The header row (and the dashes line under it) are required. Exactly how many columns to use depends on the section — see Section 6.

HTML block (the Team page only)

For laying out team member cards, use + → HTML and paste the prepared block. The exact code is in Section 5 under Team page.

Events, newsletters & reports

These are time-based lists. They each have their own listing page, generated from the routes file (Section 8).

Add an event

  1. Posts → New post. Type the event title.
  2. Add the description and any photos in the body. Keep the first paragraph short — only the first few words show on the card.
  3. Open Post settings → Tags and add event.
  4. In Excerpt, put the event date in the format YYYY-MM-DD (for example 2026-08-15). This is what sorts the event into Upcoming / Ongoing / Past.
  5. In Facebook title, paste the registration link (the "Register" button uses it).
  6. Set a Feature image and click Publish.
Do not set a future publish date for an upcoming event — Ghost would hide it until that date. Publish it now; the event date lives in the Excerpt. The page reads that date to place the event under Upcoming, Ongoing, or Past.
Past events automatically hide the "Register" button and show a "Completed" status instead — on both the events list and the event's own page. You do not need to do anything.

Add a newsletter or annual report

  1. Posts → New post. Type the title (e.g. the month and year).
  2. In the body, click + → File and upload the PDF. The card automatically shows the PDF's first page as the cover — no feature image needed.
  3. Add the tag: newsletter for monthly newsletters, or annual-report for annual reports.
  4. The publish date determines the year shown in the year filter, and the order. Click Publish.

Newsletter subscribe popup

A blue Subscribe to newsletter button sits on the Newsletters and Annual Reports pages. Clicking it opens a popup where a visitor enters their email. First-time visitors to the home page also see this popup once (it won't nag returning or already-subscribed visitors).

This is powered by Ghost's built-in Members + email system, which must be switched on once. The one-time setup (Membership, newsletter, and email delivery) is in Section 10. Until that is done, the form will show an error when submitted.

The PDF issues listed on /newsletters/ are the archive. The subscribe popup builds your email list so future posts can be emailed out — the two are separate.

Add a blog post

  1. Posts → New post. Write the title and the article in the body.
  2. Add the tag blog. The post's first tag is shown as its category.
  3. Set a Feature image and an Excerpt (the card summary), then Publish.

Add a success story (with optional video)

  1. Posts → New post. Title = the person's or story's name. Write the story in the body.
  2. Add the tag success-stories. Add a programme tag too (e.g. sevadhama-stories) to also show it on that programme page.
  3. Set a Feature image and a short Excerpt for the card.
  4. For a video story: paste a YouTube link on its own line in the body. A play button appears on the card; it plays in a popup, and "Read story" still opens the full post.
  5. Publish.

Forms & email setup

How the Contact form, the Enquire form, and the Newsletter work behind the scenes — and the one-time setup each needs.

Contact form & Enquire form

Both forms use your device's email app. When a visitor presses Submit, their email program (Gmail, Outlook, etc.) opens with a new message pre-filled from the form fields, addressed to your contact email. The visitor then presses Send in their mail app.

PointDetail
Setup neededJust the Contact email in theme settings. Nothing else.
Where it goesStraight to your contact inbox as a normal email.
Good to knowIt relies on the visitor having an email app set up. On phones this usually works; on some desktops without a default mail app, the button may do nothing.
Want submissions stored and emailed automatically (no dependence on the visitor's mail app)? Connect a free form service such as Formspree or Web3Forms — it's a one-line change to each form. Ask your developer to point the forms at the service's address; everything else stays the same.

Newsletter — how it works

  1. A visitor enters their email in the subscribe popup. Ghost creates a member and emails them a confirmation link (no password).
  2. When they click the link, they become a confirmed subscriber to your newsletter.
  3. To send an issue, you write a normal Post and choose "Publish + Email" — Ghost emails it to all subscribers. (The PDF issues on the Newsletters page are a separate archive.)

Newsletter — one-time setup in Ghost Admin

  1. Settings → Membership → turn Members on and allow free sign-ups. Without this, the subscribe popup shows an error.
  2. Settings → Email newsletter → create and name your newsletter (e.g. "Sevabharti Newsletter"), and set the sender name and reply-to address.
  3. Connect email delivery (Mailgun). Ghost sends newsletter emails through Mailgun. Create a Mailgun account, verify your sending domain, and paste the Mailgun API key, domain and region into Settings → Email newsletter. This is what actually delivers the emails.
  4. Optional: brand the sign-up/confirmation experience under Settings → Portal.
Hosting note. On Ghost(Pro), member sign-ups and confirmation emails work out of the box; you still add Mailgun to send newsletter issues. On self-hosted Ghost, configure both the transactional email (SMTP) and Mailgun.
Until Membership + Mailgun are set up, the subscribe button and the first-visit popup will appear but submitting an email will fail. Enable them first, then test with your own email.
